NovaLog Project Plan: Docking & Multi-Window MigrationIntegrating a professional docking framework is the logical next step to evolve NovaLog into a "TradingView-tier" forensic tool. This migration will solve the window management math, enable "Explode to Multi-Monitor" workflows, and provide a native home for the new Pane-Specific Follow & Sync controls.1. Migration GoalsModular Architecture: Replace the static Grid layout with a dynamic DockControl.Multi-Monitor Support: Enable "Tearing out" tabs into standalone, headless windows that maintain active log streams.State Persistence: Automatically save and restore the user's window layout (sizes, positions, and docked states) to layout.json.Deep Space Aesthetic: Skin all docking chrome (splitters, tabs, and drag handles) to match the Neon Cyan / Dark theme.2. Technical StackLibrary: Dock.Avalonia (v11.x compatible).Pattern: Factory-based MVVM (Separates layout logic from LogViewViewModel content).Window Mode: ExtendClientAreaToDecorationsHint="True" with NoChrome for a unified custom title bar.3.
